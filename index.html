Original Image

The initial photograph provided by the user.

Face Detection

The system identifies the location and boundaries of a human face within the image.

Landmark Extraction

Key facial features (eyes, nose, mouth, jawline) are mapped to precise points.

Embedding Generation

A unique mathematical vector (face descriptor) is created from the landmarks and facial structure.

Face Matching

Two facial embeddings are compared mathematically to determine their similarity.

What Might Be Stored?

  • ✓ Facial embedding (the mathematical vector)
  • ✓ User ID (to link embedding to an identity)
  • ✓ Confidence score (how sure the system is about the match)
  • ✗ Full image (not always required for recognition itself)

Disclaimer: Some real-world systems store both images and embeddings for various purposes (e.g., verification, re-training). This demo illustrates how primary recognition can be performed using embeddings alone, highlighting the non-image data aspect of facial recognition. The image data actually gets converted to a numerical representation (the embedding), which is what's primarily used for identification and comparison.
